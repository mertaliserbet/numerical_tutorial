Numerical Limits and Error Analysis in Computers

1. How Are Numbers Actually Represented in Computers?

Computers cannot represent numbers with infinite precision. Real numbers (irrational, fractional, or infinite-decimal values) are stored approximately in digital systems. Every number in a computer’s memory is represented using a specific number of bits. This bit count is fixed (typically 32 or 64), which introduces finite precision.

Thus, all mathematical operations performed by a computer involve very small differences due to these limitations, which can become critical depending on the context.

2. Floating-Point Numbers (Floating-Point Representation)

Floating-point numbers are the standard way to represent decimal numbers. The IEEE 754 standard is widely used in modern computers and programming languages (Python, C, Java, etc.). A floating-point number is divided into 3 main parts:

  • Sign (1 bit): Indicates whether the number is positive or negative.
  • Exponent (11 bits): Adjusts the magnitude of the number.
  • Mantissa/Significand (52 bits): Determines the precision by storing the significant digits.

For example, the number 123.45 can be written in scientific notation as 1.2345 x 10². The same logic applies to binary scientific notation in computers.

However, there are some limitations:

  • Very small or very large numbers may be represented inaccurately.
  • Irrational numbers (like π, e) and repeating fractions (like 1/3) cannot be represented exactly.

3. Rounding Errors

Computers often store numbers approximately, not exactly. This leads to rounding errors. While a single operation may not be noticeably affected, errors can accumulate across many operations, leading to incorrect results. 

>>> 0.1 + 0.2 == 0.3
False
>>> 0.1 + 0.2
0.30000000000000004

Additional Example:

>>> 1.1 - 1.0
0.10000000000000009
>>> 0.3 / 0.1
2.9999999999999996

4. Error Accumulation

For example, if you add 0.1 one thousand times:
Expected: 0.1 * 1000 = 100.0
Actual: 0.1 + 0.1 + ... ≈ 99.9999999999991

This can cause significant inaccuracies in iterative algorithms. To prevent this, you should use functions like math.fsum() designed for accurate summation.

5. Catastrophic Cancellation

This occurs when subtracting two numbers that are very close in value: 

x1 = 1.2345678
x2 = 1.2345677
x1 - x2 = 0.0000001

The original numbers have 8 digits of precision, but the result only has 1. This means the significant digits are lost, which can severely affect further calculations.

6. Best Practices

  • Use float64 (double precision) for precise calculations.
  • Avoid direct equality checks (==) with floating-point numbers; use numpy.isclose() for approximate comparisons.
  • Reformulate expressions to reduce cancellation. E.g., instead of √(x+1) - √(x), use 1 / (√(x+1) + √(x)).
  • Use math.fsum() when summing many numbers to minimize accumulated error.

7. Conclusion

“Why does a computer say 0.1 + 0.2 ≠ 0.3?”

Understanding these limitations is essential to designing algorithms that account for error and to writing reliable numerical software. This topic serves as a foundation for accurately implementing all numerical methods that follow.
